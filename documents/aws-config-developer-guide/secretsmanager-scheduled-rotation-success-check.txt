secretsmanager-scheduled-rotation-success-check
Checks and verifies whether AWS Secrets Manager secret rotation has rotated successfully as per the rotation schedule. The rule is NON_COMPLIANT if RotationOccurringAsScheduled is false. 
Note
The rule marks secrets without rotation NOT_APPLICABLE.
Identifier: SECRETSMANAGER_SCHEDULED_ROTATION_SUCCESS_CHECK
Trigger type: Periodic
AWS Region: All supported AWS Regions except Africa (Cape Town) and Europe (Milan)
Parameters:
None  
AWS CloudFormation template
To create AWS Config managed rules with AWS CloudFormation templates, see Creating AWS Config Managed Rules With AWS CloudFormation Templates.
