Supported Instance Types
Amazon ES supports the following instance types. Not all Regions support all instance types. For availability details, see Amazon Elasticsearch Service Pricing.
For information about which instance type is appropriate for your use case, see Sizing Amazon ES Domains, EBS Volume Size Limits, and Network Limits.




Instance Type
Restrictions




C4



C5
The C5 instance types require Elasticsearch version 5.1 or later.


I2



I3
The I3 instance types require Elasticsearch version 5.1 or later and do not support EBS storage volumes.


M3
The M3 instance types do not support encryption of data at rest, fine-grained access control, or cross-cluster search. The M3 instance types have additional restrictions by Elasticsearch version. To learn more, see Invalid M3 Instance Type.

M5
The M5 instance types require Elasticsearch version 5.1 or later.


R3
The R3 instance types do not support encryption of data at rest or fine-grained access control.
