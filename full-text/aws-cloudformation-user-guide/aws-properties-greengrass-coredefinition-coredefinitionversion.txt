AWS::Greengrass::CoreDefinition CoreDefinitionVersion
 A core definition version contains a Greengrass core.
Note
After you create a core definition version that contains the core you want to deploy, you must add it to your group version. For more information, see https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/aws-resource-greengrass-group.html.
 In an AWS CloudFormation template, CoreDefinitionVersion is the property type of the InitialVersion property in the  AWS::Greengrass::CoreDefinition  resource.
Syntax
To declare this entity in your AWS CloudFormation template, use the following syntax:
JSON
{
  "[Cores](#cfn-greengrass-coredefinition-coredefinitionversion-cores)" : [ Core, ... ]
}
YAML
[Cores](#cfn-greengrass-coredefinition-coredefinitionversion-cores): 
    - Core
Properties
Cores 
The Greengrass core in this version. Currently, the Cores property for a core definition version can contain only one core.
Required: Yes
Type: List of Core
Update requires: Replacement
