AWS::S3::Bucket MetricsConfiguration
Specifies a metrics configuration for the CloudWatch request metrics (specified by the metrics configuration ID) from an Amazon S3 bucket. If you're updating an existing metrics configuration, note that this is a full replacement of the existing metrics configuration. If you don't include the elements you want to keep, they are erased. For examples, see AWS::S3::Bucket. For more information, see  PUT Bucket metrics in the Amazon Simple Storage Service API Reference.
Syntax
To declare this entity in your AWS CloudFormation template, use the following syntax:
JSON
{
  "[Id](#cfn-s3-bucket-metricsconfiguration-id)" : String,
  "[Prefix](#cfn-s3-bucket-metricsconfiguration-prefix)" : String,
  "[TagFilters](#cfn-s3-bucket-metricsconfiguration-tagfilters)" : [ TagFilter, ... ]
}
YAML
[Id](#cfn-s3-bucket-metricsconfiguration-id): String
  [Prefix](#cfn-s3-bucket-metricsconfiguration-prefix): String
  [TagFilters](#cfn-s3-bucket-metricsconfiguration-tagfilters): 
    - TagFilter
Properties
Id 
The ID used to identify the metrics configuration. This can be any value you choose that helps you identify your metrics configuration.
Required: Yes
Type: String
Update requires: No interruption
Prefix 
The prefix that an object must have to be included in the metrics results.
Required: No
Type: String
Update requires: No interruption
TagFilters 
Specifies a list of tag filters to use as a metrics configuration filter. The metrics configuration includes only objects that meet the filter's criteria.
Required: No
Type: List of TagFilter
Update requires: No interruption
