Configuring auditing using the console
You can configure Amazon Redshift to create audit log files and store them in S3.
Enabling audit logging using the console
Note
A new console is available for Amazon Redshift. Choose either the New console or the Original console instructions based on the console that you are using. The New console instructions are open by default.
New console
To enable audit logging for a cluster


Sign in to the AWS Management Console and open the Amazon Redshift console at https://console.aws.amazon.com/redshift/.


On the navigation menu, choose CLUSTERS, then choose the cluster that you want to update. 


Choose the Maintenance and monitoring tab. Then view the Audit logging section. 


Choose Edit tab.


On the Configure audit logging page, choose to Enable audit logging and enter your choices regarding where the logs are stored. 


Choose Confirm to save your choices. 


Original console
To enable audit logging for a cluster


Sign in to the AWS Management Console and open the Amazon Redshift console at https://console.aws.amazon.com/redshift/.


In the navigation pane, choose Clusters.


In the list, choose the cluster for which you want to enable logging.


In the cluster details page, choose Database, and then choose Configure Audit Logging.


In the Configure Audit Logging dialog box, in the Enable Audit Logging box, choose Yes.


For S3 Bucket, do one of the following:
    If you already have an S3 bucket that you want to use, select Use Existing and then select the bucket from the Bucket list.

    If you need a new S3 bucket, select Create New, and in the New Bucket Name box, type a name.



(Optional) For S3 Key Prefix, enter a prefix to add to the S3 bucket.


Choose Save.


After you configure audit logging, the Cluster details page updates to display information about the logging configuration. 

On the Cluster details page, under Backup, Maintenance, and Logging, choose Go to the S3 console to navigate to the bucket. 

Modifying the bucket for audit logging
(Original console) Modifying the Amazon S3 bucket for audit logging


Sign in to the AWS Management Console and open the Amazon Redshift console at https://console.aws.amazon.com/redshift/.


In the navigation pane, choose Clusters.


In the list, choose the cluster for which you want to modify the bucket used for audit logging.


In the cluster details page, choose Database, and then choose Configure Audit Logging.


For S3 Bucket, select an existing bucket or create a new bucket.


(Optional) For S3 Key Prefix, enter a prefix to add to the S3 bucket.


Choose Save.


Disabling audit logging using the Console
(Original console) Disabling audit logging


Sign in to the AWS Management Console and open the Amazon Redshift console at https://console.aws.amazon.com/redshift/.


In the navigation pane, choose Clusters.


In the list, choose the cluster for which you want to disable logging.


In the cluster details page, choose Database, and then choose Configure Audit Logging.


In the Configure Audit Logging dialog box, in the Enable Audit Logging box, choose No.



Choose Save.
