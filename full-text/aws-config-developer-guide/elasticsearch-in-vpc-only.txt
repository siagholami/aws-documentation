elasticsearch-in-vpc-only
Checks whether Amazon Elasticsearch Service (Amazon ES) domains are in Amazon Virtual Private Cloud (Amazon VPC). The rule is NON_COMPLIANT if the Amazon ES domain endpoint is public.
Identifier: ELASTICSEARCH_IN_VPC_ONLY
Trigger type: Periodic
AWS Region: All supported AWS Regions
Parameters:
None  
AWS CloudFormation template
To create AWS Config managed rules with AWS CloudFormation templates, see Creating AWS Config Managed Rules With AWS CloudFormation Templates.
